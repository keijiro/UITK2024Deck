このように便利な UI Toolkit ですが、残念ながら現状では幾つかの制限も存在しています。
まず一つ目は、ワールドスペース UI に非対応であることです。
ワールドスペース UI というのは、要するに 3D 空間上に表示される UI のことですね。
従来の Unity UI を使った場合は、このようにワールドスペース上に UI を配置して、カメラからの視点でそれを表示することが出来ました。
このような使い方が現状の UI Toolkit では出来ません。
二つ目の制限は、カスタムシェーダー、カスタムマテリアルが使えないことです。
Unity UI では、カスタムシェーダーを用意することで、UI の見た目をカスタマイズすることが可能でした。
ですが、現状の UI Toolkit では、エレメントにマテリアルの設定を行うことが出来ません。
カスタムコントロールを実装することで見た目のカスタマイズを行うことは可能ですが、シェーダーを使った複雑なマテリアル表現は出来ない、と理解する必要があると思います。
三つ目の制限は、アニメーションクリップが使えないことです。
残念ながら、UI Toolkit では、UI エレメントに対してアニメーションクリップを使って動きを与える、ということが出来ません。
このセッションの最初のサンプルで transition animatioin を使いましたが、UI Toolkit で用意されているアニメーションの仕組みはそれだけです。
アニメーションカーブを使って細やかな動きを与えるというようなことは、現状出来ません。
もし、皆さんの開発するアプリケーションにおいてこれらの要素が重要になるようでしたら、従来の Unity UI システムを使用する必要があります。
逆にこれらの制限が問題にならないようであれば、UI Toolkit は選択肢になり得ます。<br>ぜひ導入の検討を行ってもらえればと思います。
また、これらの制限についても順次解決していくことが予定されています。<br>進捗に興味のある方は、UI Roadmap のページをチェックしてみてください。
