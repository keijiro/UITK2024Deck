今日のこのセッションでは Unity の新しい UI システムである UI Toolkit の紹介をしていきます。
よろしくお願いします。
UI Toolkit の話を始める前に、まず最初に、Unity の UI システムの全体像についておさらいしておきましょう。
というのも、恐らく皆さんご存知だと思いますが、Unity の UI システムにはいくつかの実装が存在しています。
IMGUI、これは Immidiate Mode GUI の略ですね。
Unity UI、これは別名 uGUI と呼ばれているものです。
そして今日紹介する UI Toolkit。
IMGUI は Unity に古くから存在するシステムで、これは今でも Editor 上の UI を実装するのに使われていたりします。
このシステムは、アプリのランタイム上で使用する UI システムとしては弱点が存在していたので、それを克服するための新しいシステムとして Unity UI が開発されました。
この Unity UI は、Unity で開発されるアプリのランタイム上で使用される UI システムとして広く認知されていると思います。
そしてこれら全てを置換して、より近代的なシステムに入れ替えよう、という壮大な計画が進んでいます。
それが UI Toolkit です。
これは今日の内容のちょっとしたネタバレになりますが、UI Toolkit はウェブ開発におけるスタイルシートやレイアウトシステムの概念を取り入れています。
それらの近代的な UI 開発の方法論が Unity 上で使えるようになるということですね。
また、これは個人的な感想になりますが、UI Toolkit は UI システムとして非常に出来が良いです。
これまでに登場した Unity 用の UI システムの中でも、最も使いやすいものに仕上がっている、と、個人的には考えています。
これを一度体験してしまうと、もう以前の Unity UI や IMGUI には戻れないというのが本音です。
Unity では UI Toolkit の開発をこの数年間進めてきました。
ここでは、まず Unity Editor の UI を刷新することが目標とされてきました。
皆さんお気づきかどうか分かりませんが、最近の Unity の Editor 機能は、実は既にほとんどが UI Toolkit で実装されています。
例えばこれは Unity 標準の Package Manager ですが、UI Toolkit Debugger を使って構造を確認することができます。
Debugger から値を書き換えることも可能です。
と、こんな感じで、UI Toolkit は既に Unity Editor 上で本格的に使用されている、実用レベルのシステムなんです。
そしてこれを Editor 上だけでなく、アプリのランタイム側で使えるようにするための対応も進められてきました。
特に Unity 6 では、Runtime Data Binding など、ランタイム時に便利に使える強力な機能が追加されています。
まだ幾つかの制約はあるのですが、ランタイム用 UI システムの選択肢のひとつとして十分にお勧めできるものになっていると思います。
まだ UI Toolkit に触れたことが無いという方は、今日のこのセッションを通して UI Toolkit のことを知っていただきたいと考えています。
さて、今日は次のように話を進めていきたいと思います。
まず最初に、Unity 公式のサンプルプロジェクトを使ったデモをお見せします。
次に、いわゆる "Hello World" ような、ごく簡単なサンプルプロジェクトを使って、実際に UI Toolkit で UI を構築する様子をお見せします。
その次に、少し高度な機能として Runtime Data Binding を使った例をお見せします。
最後に、現状における UI Toolkit の制限についてお話しします。
